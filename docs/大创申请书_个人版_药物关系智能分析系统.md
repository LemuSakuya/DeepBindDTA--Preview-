# 大学生创新创业训练计划项目申请书（草案）

## 一、项目名称

面向自然语言交互的药物关系智能分析系统研究与实现

## 二、项目类别

创新训练项目

## 三、项目简介

本项目面向药物研发与用药安全分析中的实际需求，拟研发一个“查询、预测、解释”一体化的药物关系智能分析系统。系统以自然语言为输入入口，能够自动识别用户问题中的药物实体、靶标实体及关系意图，在已知知识库中优先完成关系检索；当知识库未覆盖时，调用深度学习模型对药物-靶标作用关系或潜在药物关系进行预测；最终以结构化结果和自然语言说明两种形式输出分析结论。

与传统仅提供检索或仅输出预测分数的工具不同，本项目强调从“用户怎么提问”到“系统如何解释结果”的完整闭环，突出交互友好性、分析可解释性和系统集成性。项目将结合现有深度学习药物-靶标亲和力预测框架、数据库查询模块、图谱关系组织思路以及自然语言解释生成方法，形成一个可演示、可扩展、可服务科研场景的软件原型。

## 四、立项背景与研究意义

随着人工智能、深度学习和大模型技术在生物医药领域的快速发展，药物发现逐渐由传统高成本、长周期的实验驱动模式向“数据驱动 + 智能辅助”的新范式转变。药物-靶标相互作用预测、药物-药物相互作用分析、药物重定位与用药风险评估等问题，已成为智能药学研究中的重要方向。尤其在候选药物筛选、药理机制推断和临床联合用药分析中，准确、高效且可解释的药物关系分析工具具有重要价值。

然而，现有系统仍存在若干明显不足。首先，多数平台依赖结构化输入，要求用户预先掌握药物编号、标准字段或特定数据库格式，这提高了使用门槛，不利于普通本科生科研训练与非专业人员使用。其次，很多工具只能完成单一任务，要么只能查已知关系，要么只能输出预测结果，缺乏“已知信息调用 + 未知关系发现”的联动机制。再次，大多数预测模型仅给出标签、分类结果或概率分值，缺少自然语言层面的解释，用户难以理解“为什么得到这样的判断”，这在药学场景下会显著影响系统的可用性与可信度。

本项目拟从真实使用场景出发，构建一个以自然语言为入口、以解释性输出为终点的药物关系智能分析系统，推动药物智能分析从“模型演示”向“科研辅助系统”转变。其研究意义主要体现在以下几个方面：

1. 有助于降低药物关系分析工具的使用门槛，提升科研交互友好性。
2. 有助于将知识检索、关系预测和自然语言解释统一到同一系统框架中，增强系统实用价值。
3. 有助于探索大模型、知识组织与深度学习预测在药学任务中的协同应用，为后续科研拓展和论文工作奠定基础。
4. 有助于形成可展示的软件原型和较完整的本科生项目成果，适合大创项目的训练目标与验收形式。

## 五、国内外研究现状

当前与本项目相关的研究主要集中在三个方向。

第一，面向药物发现任务的智能体与自然语言系统研究正在快速发展。相关工作表明，大模型不仅能够完成问答，还可以通过任务分解、工具调用和多轮推理参与药物研发流程。此类研究为本项目中的自然语言解析、任务调度和模块联动提供了方法参考，也说明“面向自然语言输入的药物智能系统”已具备现实可行性。

第二，药物关系预测研究已形成较成熟的技术基础。围绕药物-靶标亲和力预测、药物-药物相互作用识别、图神经网络建模、多模态特征融合等方向，国内外已积累大量方法与数据资源。这些研究为本项目的预测模块提供了可继承的模型思路，尤其是基于分子表示、蛋白表示与深度神经网络的关系推断方法，能够为系统中的未知关系分析提供较高质量的候选结果。

第三，药物关系结果的自然语言解释研究逐渐受到重视。最新工作开始将解释生成视为独立任务，不再满足于只输出关系类别，而是进一步生成机制描述、证据归纳和关系说明文本。这说明药物智能分析系统的价值已从“给出答案”延伸到“解释答案”，也为本项目构建解释性输出模块提供了明确方向。

总体而言，国内外研究虽然已分别在任务调度、关系预测与解释生成方面取得进展，但仍缺少一个真正以自然语言用户交互为中心、统一整合“输入解析、知识查询、模型预测、结果解释”的端到端系统。本项目拟在现有工作基础上完成模块化整合与原型实现，形成适合本科阶段创新训练的研究成果。

## 六、项目目标

本项目拟实现以下目标：

1. 构建面向自然语言问题的药物关系任务解析模块，实现药物实体、靶标实体及关系意图识别。
2. 设计已知关系优先检索、未知关系自动预测的双通路分析机制。
3. 在现有药物-靶标预测模型基础上，完成与系统前端和数据库模块的联动集成。
4. 构建面向查询结果与预测结果的自然语言解释生成模块，增强系统可读性与可解释性。
5. 开发一个具备交互界面或演示能力的软件原型，并通过典型案例验证系统有效性。

## 七、研究内容

### 1. 自然语言输入解析与实体标准化

围绕用户输入的自然语言问题，研究药物名称、蛋白靶标名称、药物对与关系意图的自动识别方法。重点解决药物别名、蛋白名称不统一、用户表达口语化等问题，将自然语言输入转换为系统可处理的结构化任务表示。

### 2. 药物关系知识查询与数据组织

基于现有数据库和本地关系资源，构建已知关系查询模块，支持药物基础信息、部分药物关系信息和靶标相关条目的快速检索。该部分不仅提供查库能力，也为后续解释生成提供基础事实和证据来源。

### 3. 未知关系预测模块构建与适配

依托已有药物-靶标亲和力预测框架，完成分子与蛋白输入处理、模型调用、结果输出与系统集成。针对知识库中未命中的场景，系统能够自动切换到预测模式，对潜在药物-靶标关系进行分值计算和结果生成。

### 4. 自然语言解释生成机制设计

针对查询结果和预测结果，构建解释模板与知识增强描述机制。对于已知关系，重点输出关系类型、来源和已有说明；对于预测结果，重点围绕预测分数、相似样本、相关靶标信息或机制线索生成解释文本，使输出更符合科研沟通需求。

### 5. 原型系统开发与案例验证

在现有软件基础上完善前后端联动、模型调用、数据库检索和结果展示流程，形成可运行的原型系统。选取典型药物、靶标和药物对开展案例测试，验证系统的完整性、稳定性和展示效果。

## 八、拟解决的关键问题

1. 如何准确识别自然语言中的药物、靶标及关系意图，并完成规范化映射。
2. 如何实现“已知关系查询”和“未知关系预测”的统一调度，避免系统逻辑割裂。
3. 如何让预测结果不仅有分值，还具备可读、可信的自然语言解释。
4. 如何在本科生可承受的工作量下完成一个真正可演示、可答辩的软件原型。

## 九、项目创新点

1. 以自然语言交互作为系统入口，突出真实科研场景下的人机交互需求，而非仅面向程序接口或表格输入。
2. 设计“查询优先、预测补充”的双通路决策机制，统一已知关系调用与未知关系发现。
3. 将解释生成纳入系统核心功能，使输出从单纯“结果给出”提升为“结果说明”。
4. 将深度学习预测、知识查询、自然语言解析和界面展示整合为一个完整系统，强调工程实现与应用落地。

## 十、技术路线

本项目总体技术路线如下：

自然语言输入
→ 实体识别与关系意图解析
→ 已知关系查询
→ 若命中知识库，则返回检索结果并生成说明
→ 若未命中知识库，则调用预测模型进行关系推断
→ 融合查询结果或预测结果生成自然语言解释
→ 输出结构化分析结果与解释文本

具体而言：

1. 输入层负责接收用户自然语言问题，并完成问题清洗、关键词抽取与实体候选识别。
2. 解析层负责药物名、靶标名和关系意图的匹配与标准化。
3. 查询层负责本地数据库或知识资源的优先检索。
4. 预测层负责调用深度学习模型对未知药物-靶标关系进行推断。
5. 解释层负责基于结果、规则、知识片段或模板生成可读文本。
6. 展示层负责通过图形界面输出分析结果、预测分值和解释内容。

## 十一、研究基础与可行性分析

本项目具有较好的研究基础。

首先，项目已有初步软件原型和技术积累。当前仓库已经具备药物-靶标亲和力预测模型、预测脚本、数据处理模块、图形界面程序和数据库连接模块，说明项目并非从零开始，而是在已有代码和功能基础上继续扩展与整合。

其次，相关数据与模型资源较为明确。项目已经包含药物、蛋白、预测样例、训练模型权重与数据库导入脚本，可支撑后续实验与系统演示。

再次，研究任务边界清晰。项目重点聚焦于自然语言输入、关系查询、未知关系预测和结果解释四个核心模块，目标明确，适合本科生在导师指导下分阶段完成。

最后，成果形式直观。系统原型、案例展示、文档材料和答辩演示均较容易组织，便于形成大创项目所需的中期检查和结题成果。

因此，从已有基础、任务拆分、技术成熟度和成果可展示性来看，本项目具备较高可行性。

## 十二、实施方案与进度安排

### 第一阶段：前期调研与方案设计（第 1-2 个月）

1. 阅读并整理与药物关系分析、自然语言智能体、解释生成相关的文献资料。
2. 结合已有项目代码，明确系统功能边界、技术路线和模块划分。
3. 完成项目申请、开题材料及总体设计文档。

### 第二阶段：查询模块与输入解析模块开发（第 3-5 个月）

1. 完成自然语言输入的实体识别、别名归一化和任务结构化表示。
2. 打通药物、蛋白和已有关系的查询流程。
3. 完成基本前端交互和结果展示逻辑。

### 第三阶段：预测模块与解释模块开发（第 6-8 个月）

1. 将现有预测模型进一步封装并接入系统主流程。
2. 设计查询失败后自动预测的逻辑切换机制。
3. 完成基于模板与知识片段的自然语言解释生成功能。

### 第四阶段：系统集成与案例验证（第 9-10 个月）

1. 完成系统联调、界面优化和异常处理。
2. 选取代表性药物-靶标或药物组合开展案例测试。
3. 对系统功能、可解释性和演示效果进行总结与完善。

### 第五阶段：成果整理与结题答辩（第 11-12 个月）

1. 完成项目总结报告、中期和结题材料整理。
2. 完成系统演示视频、PPT 和典型案例说明。
3. 在条件允许的情况下形成综述、论文初稿或软著材料。

## 十三、预期成果

1. 一个面向自然语言交互的药物关系智能分析原型系统。
2. 一套“输入解析、关系查询、模型预测、结果解释”一体化的技术方案。
3. 若干典型药物关系分析案例，用于支撑答辩演示和效果验证。
4. 项目申请书、中期检查材料、结题报告等完整文档。
5. 在条件成熟的情况下，形成论文初稿、软件著作权申请材料或竞赛支撑材料。

## 十四、项目风险与应对措施

1. 若自然语言实体识别效果不稳定，则采用“规则 + 候选匹配 + 人机确认”的方式降低误识别风险。
2. 若知识库覆盖不足，则将系统重点放在药物-靶标预测与可解释展示上，确保核心功能可用。
3. 若解释生成效果较弱，则优先采用模板化解释和证据拼接方式，保证输出稳定性。
4. 若模型推理速度影响演示体验，则通过缓存结果、限制演示样例规模和提前加载模型等方式优化响应速度。

## 十五、指导教师意见可支撑点

从项目定位看，本课题兼具研究性与工程实现价值，既能够训练学生在文献调研、系统设计、模型应用、数据处理和软件开发方面的综合能力，也具有明确的成果呈现形式。项目难度适中、路线清晰、已有基础较好，符合大学生创新训练项目的培养目标。

## 十六、结论

本项目围绕药物关系智能分析中的“难输入、难联动、难解释”三个实际问题，提出构建面向自然语言交互的药物关系智能分析系统。项目将现有深度学习预测技术、数据库查询能力和自然语言解释机制进行有机融合，既具有一定研究创新性，也具备较强的实现基础与展示价值。项目预期能够形成一个较完整的软件原型，并为后续深入研究、论文写作和竞赛拓展打下基础。
