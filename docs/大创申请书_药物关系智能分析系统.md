# 面向自然语言输入的药物关系智能分析系统研究

## 一、项目背景与研究意义

随着人工智能在药物研发中的广泛应用，药物-药物相互作用和药物-靶标关系预测逐渐成为药物发现、药物重定位和用药安全分析中的重要研究方向。现有研究虽然在关系预测、知识检索和智能体调度等方面取得了一定进展，但大多仍存在三个明显不足：一是用户交互方式偏结构化，通常依赖表格、固定字段或专业输入格式，不利于面向普通科研使用者；二是系统往往只能完成“查询”或“预测”中的单一功能，缺乏对已知关系检索与未知关系发现的统一处理机制；三是输出形式多停留在标签、分数或关系类别层面，缺乏对关系机制和预测依据的自然语言解释，难以满足科研辅助与结果展示需求。

导师提出的核心要求是，系统应以自然语言为起点，以自然语言解释为终点。也就是说，用户输入一句自然语言，例如“帮我查一下药物 A 和药物 B 的关系”或“预测某药和某靶标是否存在作用”，系统应先自动识别其中的药物实体、药物对或药物-靶标对，再判断关系是否已存在于知识库中。若已存在，则返回查询结果；若不存在，则调用模型进行预测；最终还应输出一段关于该关系的描述性语句，说明关系类型、可能机制或系统判断依据。

因此，本项目拟构建一个“查询-预测-解释”一体化的药物关系智能分析系统，实现从自然语言输入到结构化关系结果和自然语言解释输出的完整闭环。这一研究不仅有助于提升药物关系分析系统的交互性和可解释性，也对构建面向药物发现的智能问答与科研辅助平台具有现实意义。

## 二、国内外研究现状

近年来，自然语言大模型、智能体系统以及知识增强生成方法的发展，为药物关系分析系统提供了新的技术基础。相关研究大致可以概括为三个方向。

第一类是面向药物发现流程的智能体系统研究。Li 等提出的 DrugPilot 强调通过参数化记忆池和反馈-聚焦机制，实现复杂药物任务的多轮工具调用和任务管理，为自然语言任务解析和系统调度提供了思路。Liu 等提出的 DrugAgent 则通过 Planner 与 Instructor 的协作，将药物发现中的方案规划、领域知识注入与代码执行组织为一个系统流程，说明自然语言智能系统不能只是“会回答”，还需要“会组织任务”。

第二类是药物关系预测研究。Inoue 等提出的 DrugAgent 面向 DTI 任务，将机器学习预测、知识图谱路径和文献检索证据整合到统一推理框架中，体现了药物关系判断从单一模型预测向多源证据融合推理的发展趋势。这说明在药物关系任务中，预测结果如果能够结合知识证据，将更具可信度。

第三类是自然语言解释生成研究。Sun 等提出的 ExDDI 首次较系统地将 DDI 解释生成作为独立任务，强调“关系预测”和“自然语言解释”是两个不同但应联合设计的目标。Wang 等提出的 PKAG-DDI 则进一步说明，高质量解释生成需要引入机制相关的成对知识，而不能停留在对结果的简单复述。

总体来看，现有研究分别在智能体调度、关系预测和解释生成方面提供了较成熟的参考，但仍缺乏一个真正面向自然语言用户输入、能够统一完成实体识别、关系查询、未知关系预测与自然语言解释的端到端系统。本项目拟解决的正是这一问题。

## 三、研究目标

本项目拟构建一个面向自然语言输入的药物关系智能分析系统，实现以下目标：

1. 实现自然语言问题解析，能够自动识别药物、靶标及其关系意图。
2. 构建统一的查询与预测双通路机制，对已知关系进行知识检索，对未知关系进行模型推断。
3. 实现关系结果的自然语言解释生成，使系统输出不仅包含关系标签和置信度，还包含对关系机制或判断依据的描述性说明。
4. 完成一个可演示的原型系统，展示从自然语言输入到结果输出的完整流程。

## 四、研究内容

### 1. 自然语言问题解析与实体识别

研究面向药物关系任务的自然语言理解方法，从用户输入中识别药物名称、靶标名称、药物对以及关系意图，并完成别名归一化和结构化任务表示。该部分重点解决“用户怎么说”和“系统怎么理解”之间的转换问题。

### 2. 已知关系查询与知识整合

构建药物关系知识查询模块，优先从已有知识资源中检索药物-药物或药物-靶标的已知关系，并返回关系类型、来源信息及已有说明。该部分不仅是简单查库，还需整合关系来源和基础证据，为后续解释生成提供支撑。

### 3. 未知关系预测

在知识库中无法命中关系时，系统自动进入预测模式，对药物-药物或药物-靶标之间的潜在关系进行推断。该模块重点服务于“知识库未覆盖”的场景，是系统完成未知关系发现的关键部分。

### 4. 自然语言解释生成

针对查询结果或预测结果生成自然语言说明。对于已知关系，重点归纳已有关系描述和机制信息；对于预测关系，重点围绕预测分数、相似性依据、知识图谱路径或候选机制生成解释性语句。该部分是导师强调的核心要求，也是本项目区别于传统预测系统的重要特征。

## 五、拟解决的关键问题

1. 如何准确识别自然语言中的药物实体、靶标实体和关系意图。
2. 如何在已知关系查询和未知关系预测之间建立统一联动机制。
3. 如何将结构化关系结果转化为可读、可信的自然语言解释。
4. 如何将自然语言理解、关系查询、关系预测与解释生成稳定整合为一个端到端系统。

## 六、技术路线

本项目拟采用“模块化设计、双通路决策、统一解释输出”的技术路线，整体流程如下：

自然语言输入  
→ 实体识别与关系意图解析  
→ 已知关系查询  
→ 若命中知识库，则直接返回关系结果并生成解释  
→ 若未命中知识库，则调用预测模块进行关系推断  
→ 基于查询结果或预测结果生成自然语言说明  
→ 输出结构化关系结果与解释文本

具体来说，自然语言解析模块主要借鉴 DrugPilot 和 Liu 等工作在任务结构化与系统调度方面的思路；关系查询与证据整合模块参考 Inoue 等多源证据融合的思想；解释生成模块重点吸收 ExDDI 关于“预测 + 解释”联合设计的理念，并参考 PKAG-DDI 的机制知识增强思想。通过这种方式，本项目不是简单复现某一篇论文，而是将五篇文章的核心思想融入一个统一系统中。

## 七、创新点

1. 以自然语言问题为系统入口，构建面向真实交互场景的药物关系分析流程。
2. 设计“查询优先、预测补充”的双通路机制，统一已知关系调用与未知关系发现。
3. 将自然语言解释生成纳入系统核心输出，使系统能够回答“是什么关系”和“为什么是这种关系”两个层次的问题。
4. 融合智能体调度、多源证据推理和机制增强生成思想，形成“自然语言输入—关系分析—自然语言输出”的完整闭环。

## 八、实施计划

### 第一阶段：前期调研与方案设计

完成文献调研和问题定义，明确系统整体框架与模块划分，梳理药物关系相关知识资源。

### 第二阶段：自然语言解析与查询模块开发

实现实体识别、别名归一化和关系查询功能，打通从自然语言输入到知识检索的基本流程。

### 第三阶段：预测与解释模块开发

构建未知关系预测模块和自然语言解释生成模块，形成“查询失败后自动预测并解释”的核心闭环。

### 第四阶段：系统集成与案例验证

完成系统集成，选取若干典型药物对和药物-靶标对进行测试，展示系统完整功能并整理结题材料。

## 九、预期成果

1. 一个面向自然语言输入的药物关系智能分析原型系统。
2. 一套“查询-预测-解释”一体化的系统技术方案。
3. 典型案例展示材料，用于验证系统功能和项目可行性。
4. 项目申请书、中期检查材料、结题报告以及后续可拓展的论文或综述文本。

## 十、可行性分析

本项目具有较好的实施基础和可行性。首先，项目问题边界清晰，聚焦于自然语言药物关系分析闭环，避免过度发散。其次，现有研究已分别在智能体调度、药物关系预测和解释生成方面提供了较成熟的技术参考，本项目可在其基础上进行模块化集成。再次，本项目强调原型系统实现与典型案例展示，适合本科生创新项目的工作量与成果形式要求。最后，项目输出形式明确，便于在答辩和验收中直观展示。
